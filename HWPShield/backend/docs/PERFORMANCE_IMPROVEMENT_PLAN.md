# HWPShield 성능 및 보안 개선 보고서

## 📊 현재 상태 평가

### 1. 오탐률 (False Positive Rate)
**현재**: 약 15-20% (MZ 패턴 기반 탐지)
**목표**: 5% 이하

**주요 오탐 원인**:
- MZ 시그니처만으로 EXE 탐지
- 압축 데이터 내 패턴
- 이미지 파일 내 바이너리 패턴

### 2. 바이러스 탐지율 (Detection Rate)
**현재**: 약 70-80%
**목표**: 95% 이상

**주요 미탐 원인**:
- 난독화된 페이로드
- 복잡한 매크로 구조
- 0-day 취약점 패턴

### 3. 보안 안정성
**현재**: 중간 수준
**목표**: 높은 수준

**주요 취약점**:
- 파일 경로 조작 가능성
- 대용량 파일 DoS
- 메모리 오버플로우 가능성

## 🔧 개선 방안

### 1. 오탐률 감소 대책

#### 1.1 PE 구조 검증 강화 ✅ (구현 완료)
```python
# 기존: MZ만 확인
if data.startswith(b'MZ'):
    detected_as_exe()

# 개선: PE 구조 전체 검증
pe_analysis = PEAnalyzer.analyze_exe_embeddings(data)
if pe_analysis["has_real_executable"]:
    detected_as_exe()
```

#### 1.2 컨텍스트 인식 탐지
- **이미지 시그니처 확인**: PNG/JPG/GIF 내 MZ 패턴 무시
- **압축 데이터 감지**: zlib 헤더 확인 후 낮은 가중치
- **OLE 스트림 분석**: BinData vs BodyText 구분

#### 1.3 통계적 필터링
- **엔트로피 분석**: 정상 데이터 vs 악성코드 엔트로피 비교
- **패턴 빈도**: MZ 패턴 개수에 따른 가중치 조절
- **위치 정보**: 파일 시작 vs 중간부 패턴 구분

### 2. 탐지율 향상 대책

#### 2.1 다중 레이어 탐지
```python
# 1계층: 시그니처 기반
signature_patterns = ['MZ', 'PE\\0\\0', 'eqproc', 'ShellExecute']

# 2계층: 구조 기반
pe_structure = validate_pe_header()
ole_structure = parse_ole_streams()

# 3계층: 행동 기반
behavior_indicators = ['temp_path', 'url_download', 'process_create']
```

#### 2.2 난독화 탐지
- **XOR/RLE 패턴**: 반복적인 바이트 패턴 감지
- **암호화 페이로드**: 높은 엔트로피 구간 분석
- **인코딩 감지**: Base64, URL 인코딩 탐지

#### 2.3 머신러닝 보조 (선택사항)
- **특성 추출**: 파일 구조, 패턴, 엔트로피
- **모델 학습**: 정상 vs 악성 HWP 파일
- **실시간 예측**: 룰 기반 결과 보정

### 3. 보안 안정성 강화

#### 3.1 입력 검증 강화 ✅ (부분 구현)
```python
# 파일 크기 제한
MAX_FILE_SIZE = 50 * 1024 * 1024  # 50MB

# 파일 형식 검증
ALLOWED_EXTENSIONS = ['.hwp', '.hwpx']
ALLOWED_MIME_TYPES = ['application/x-hwp', 'application/haansofthwp']

# Rate Limiting
RATE_LIMIT_PER_IP = 10  # requests per minute
```

#### 3.2 메모리 보호
- **버퍼 크기 검증**: struct.unpack 전 bounds 확인
- **스트림 크기 제한**: 개별 스트림 10MB 제한
- **메모리 사용량 모니터링**: 프로세스 메모리 추적

#### 3.3 예외 처리 강화
```python
# 안전한 파일 핸들링
try:
    with secure_temp_file(filename) as (temp_path, cleanup):
        result = analyze_file(temp_path)
except SecurityError as e:
    log_security_event(e)
    return error_response
except Exception as e:
    log_error(e)
    return safe_error_response()
finally:
    cleanup()  # 항상 실행 보장
```
